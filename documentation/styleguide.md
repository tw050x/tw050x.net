# Styleguide

This document outlines the coding style and best practices for our codebase. Adhering to these guidelines ensures consistency, readability, and maintainability across all projects.

See the below rules for more information.

## Rules (in no particular order)

### Lines should always start with a letter.

You should always start lines of code with a letter (a-z, A-Z). This improves readability and makes for a pleasant experience when scanning through code.

Lines that are for code comments, are closing brackets or braces, or are part of a key-value pair in objects that need to be quoted or contained in square brackets are excluded from this rule.

Spread operations and ternary operators are also excluded from this rule.

Lines beginning with JSX Elements in embedded JavaScript (JSX) are also excluded from this rule.
